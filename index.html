Document Object Model

Co je to DOM?

Zkratkou DOM označujeme objektový model dokumentu (anglicky Document Object Model). Jedná se o rozhraní, přes které mohou programátoři provádět změny struktury, obsahu i stylu HTML stránky.

DOM reprezentuje webovou stránku jako model tvořený uzly (nodes) a objekty (objects). Díky tomu můžeme využívat principů objektově orientovaného programování a pomocí JavaScriptu manipulovat s jakýmkoliv prvkem stránky. Dříve statický web se tak stává dynamickým a zároveň interaktivním, protože uživatel prostřednictvím myši, klávesnice, v případě mobilních zařízení i prsty, vyvolává na stránce nejrůznější akce - v objektovém programování označované jako události (events). V okně prohlížeče tak mohou být spouštěny plnohodnotné aplikace, které jsou v mnoha případech rovnocenné programům nainstalovaným v lokálních počítačích a běžících v grafických operačních systémech.

Standard W3C DOM (WHATWG DOM) je implementován ve většině moderních prohlížečů. Řada z nich však tento standard rozšiřuje vlastním specifickým způsobem, což pochopitelně znesnadňuje práci programátorům, kteří s tím musejí počítat při tvorbě svého kódu, chtějí-li zajistit maximální kompatibilitu svým webovým aplikacím.

Všechny vlastnosti (properties), metody (methods) a události (events) určené pro manipulaci a vytváření webových stránek jsou organizovány do objektů. Například základem celé stránky je objekt document (tedy dokument HTML), tabulku zase tvoří speciální HTMLTableElement DOM rozhraní, specifickým objektem je i obrázek a podobně.

Moderní DOM tvoří vícero APIs (Application Programming Interfaces), tedy různých programových knihoven, s jejichž pomocí programátoři sestavují vlastní webové aplikace. K nejdůležitějším patří HTML DOM API umožňující přístup k běžným objektům webové stránky (nadpisům, odstavcům, odkazům, seznamům, tabulkám, obrázkům apod.). K základním dovednostem programátora webových aplikací proto patří schopnost dobře se orientovat v dokumentacích a referencích API.

DOM a JavaScript

DOM není programovací jazyk; představuje skutečně jen strukturu dokumentu převedenou do podoby objektového modelu, která by měla být nezávislá na konkrétním programovacím jazyku. K objektům stránky díky tomu můžeme přistupovat s využitím různých programovacích jazyků, například Pythonu.

Přesto se zdaleka nejčastěji používá pro účely manipulace s objekty webové stránky skriptovací jazyk JavaScript. Spojením JavaScriptu s DOM vzniklo několik API a ty budeme využívat při tvorbě našich webových aplikací.

K objektům stránky lze přistupovat po jejich načtení do okna prohlížeče. I z toho důvodu je vhodnější, když skripty do kódu HTML začleníme až v samotném závěru - tedy před ukončením těla stránky. Nejpřehlednější a nejpraktičtější je připojení samostatných javascriptových souborů, abychom nemíchali jazyk HTML se skriptovacím jazykem.

Základním objektem DOM je objekt document, který zahrnuje samotnou webovou stránku a všechny v ní obsažené elementy.

Dalším důležitým objektem je window, představující okno prohlížeče. Součástí objektu window jsou mimo jiné i metody, jimiž můžeme vyvolat "vyskakovací" popup okna (window.alert(), window.prompt() nebo window.confirm()).

Přehled základních datových typů (rozhraní) DOM

Datový typ (rozhraní) Popis
document Základní, kořenový (root) objekt představující dokument (HTML nebo XML).
node Každý objekt umístěný v dokumentu je určitým typem uzlu (node). Uzlem je v HTML dokumentu nejen libovolný element (element node), ale rovněž vnitřní text (text node) nebo atribut (atribut node).
element Element (tedy jakákoliv značka v HTML) je speciálním typem uzlu v dokumentu (tedy na webové stránce). Metodou document.createElement() můžeme na stránce vytvořit nový element.
nodeList Objekt nodeList představuje pole (seznam) elementů či uzlů, které můžeme získat například použitím metody document.querySelectorAll(). Jednotlivé položky nodeList jsou přístupné následujícími dvěma způsoby:
  • list.item(1)
  • list[1]
Číslo 1 zde představuje tzv. index určující pořadí elementu (uzlu) v poli (seznamu). Protože počáteční index je 0, jde v tomto případě o odkaz na druhý prvek pole (seznamu).
attribute Speciální rozhraní pro atributy, které mohou být přiřazeny k HTML elementům (např. class, src, title atd.).
namedNodeMap Rozhraní umožňující přistupovat k poli (seznamu) atributů určitého uzlu pomocí jejich názvů.
